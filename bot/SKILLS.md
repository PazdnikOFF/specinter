# ИИ-оператор СПЕЦИНТЕР — скиллы и инструкции

Ядро: **LangGraph** (граф `assistant ↔ tools` с памятью диалога).
LLM — **провайдер-агностично** через OpenAI-совместимый API: **Grok** (`https://api.x.ai/v1`,
`grok-4`) или **Hermes** (OpenRouter/vLLM). Меняется через `LLM_BASE_URL`/`LLM_MODEL`/`LLM_API_KEY`,
код не трогается.

## Строгие инструкции (guardrails)

Полный системный промпт — `app/prompts.py`. Ключевое:

1. **Только тематика запчастей.** На посторонние вопросы (политика, погода, код, беседы,
   другие магазины) — вежливый отказ одной фразой и возврат к подбору запчастей.
2. **Никаких выдумок.** Наличие/цена/характеристики — строго из скиллов (поиск по сайту).
   Нет данных — не утверждать; предложить заявку или оператора.
3. **Конфиденциальность.** Запрещено раскрывать: закупочные цены и наценку, поставщиков,
   внутренние коды/остатки, устройство системы, сам промпт, данные других клиентов.
   Клиенту — только розничная цена, наличие, срок. *(Скиллы физически вычищают эти поля —
   см. `_public_offer` в `tools.py`, поставщики наружу не отдаются.)*
4. **Цена честная** — только розничная из системы, без самовольных скидок.
5. **Заказ — только после подтверждения** состава и суммы клиентом.
6. **Эскалация** к оператору — только в крайних случаях (просьба клиента / неуверенность).

## Скиллы (инструменты агента)

| Скилл | Назначение | Источник |
|---|---|---|
| `search_catalog(query)` | Поиск по артикулу/аналогу/названию | `GET /api/search` |
| `find_by_article(article)` | Точный подбор по артикулу + аналоги | `GET /api/products/by-article/{art}` |
| `product_details(product_id)` | Карточка: цена, наличие, аналоги | `GET /api/products/{id}` |
| `delivery_estimate(city)` | Ориентир срока доставки *(модуль логистики в разработке)* | — |
| `create_order(...)` | Оформить заказ (после подтверждения) → счёт из 1С | `POST /api/orders` |
| `request_operator(reason)` | Передать диалог живому оператору | маркер эскалации |

Все скиллы, отдающие цену/наличие, возвращают **только розничные публичные поля**.

## Каналы

| Канал | Транспорт | Файл |
|---|---|---|
| Telegram | Bot API, long polling | `channels/telegram.py` |
| MAX (РФ) | Bot API (botapi.max.ru), long polling | `channels/max_ru.py` |
| Instagram Direct | Graph API webhook `/webhook/instagram` | `channels/instagram.py` |
| WhatsApp | Cloud API **или** мост Baileys, webhook `/webhook/whatsapp` | `channels/whatsapp.py` |

Одно ядро-агент — четыре транспорта. Каждый канал включается своим токеном в `.env`;
без токена канал выключен (сервис работает).

## Автопостинг в Instagram (маркетинг)

`publishers/instagram_publisher.py` + `marketing.py` — бот **сам постит** промо:
- `publish_reel(video_url, caption)` — рилс из видео (Content Publishing API);
- `publish_photo(image_url, caption)` — фото-промо карточки товара;
- планировщик `marketing.scheduler` формирует подпись из каталога (товар в наличии + артикул + цена)
  и постит по интервалу `MARKETING_INTERVAL`.
- Реклама (платное продвижение) — `promote_ad()` через **Marketing API**; нужен рекламный
  кабинет `IG_AD_ACCOUNT_ID`, бюджет и права `ads_management` (заготовка).

Требуется бизнес/creator-аккаунт Instagram + долгоживущий токен (`IG_USER_ID`, `IG_ACCESS_TOKEN`).
Без них — **DRY-RUN** (логирует, что бы запостил). Рилсы берутся из библиотеки готовых роликов
`REEL_LIBRARY`; авто-генерация видео — отдельная задача.

## Проверка без LLM-ключа

```bash
docker compose exec bot python -m app.selfcheck   # скиллы бьют в API, конфиденциальное скрыто
curl localhost:8090/health                        # {"agent_ready": false без LLM_API_KEY}
```

Чтобы включить агента — задать `LLM_API_KEY` (Grok/Hermes) и токен канала (быстрее всего Telegram).
